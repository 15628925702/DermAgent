# Root Organization Guide

This file explains what belongs in the project root and what should live elsewhere.

## Keep In Root

These are the minimal root-level entry points and project metadata:

- `agent/`, `skills/`, `memory/`, `datasets/`, `evaluation/`, `integrations/`, `models/`, `llm/`, `config/`
- `scripts/`
- `tests/`
- `data/`
- `outputs/`
- `docs/`
- `legacy/`
- `README.md`
- `requirements.txt`
- `pytest.ini`
- `.gitignore`

## Code Directories

- `agent/`: orchestration and diagnosis flow
- `skills/`: executable skills and specialists
- `memory/`: experience bank, retrieval, compression, writeback
- `datasets/`: dataset loaders and split helpers
- `evaluation/`: metric and comparison helpers
- `integrations/`: external adapters
- `models/`, `llm/`, `config/`: model and config glue

These should stay under version control.

## Data

- `data/pad_ufes_20/`: dataset files

Keep the dataset under `data/`. Do not scatter data files at the root.

## Outputs And Artifacts

Everything generated by training, evaluation, testing, or local services should go under `outputs/`.

Recommended layout:

- `outputs/checkpoints/seed/`: tracked seed controller and seed memory bank
- `outputs/train_runs/`: training checkpoints and summaries
- `outputs/archive/`: archived old runs and old backup checkpoints
- `outputs/splits/`: dataset split manifests
- `outputs/logs/`: local service or manual run logs
- `outputs/test_artifacts/`: pytest temp files and cache

If a new script writes generated files, it should default to a subdirectory of `outputs/`.

## Logs

Logs should not be written to the project root.

Use:

- `outputs/logs/qwen_vllm.log`
- `outputs/logs/train_8h.log`

The default vLLM startup script has been updated to use `outputs/logs/`.

## Safe To Delete

Usually safe to delete when you do not need them:

- `outputs/test_artifacts/`
- `outputs/logs/`
- old run directories under `outputs/train_runs/`
- `__pycache__/` directories

Delete carefully:

- `outputs/checkpoints/seed/controller_v2.json`
- `outputs/checkpoints/seed/experience_bank.json`

These may be checkpoints you still care about.

## Should Not Be Reintroduced At Root

Avoid putting these directly under the root:

- ad hoc `*.log` files
- pytest temp directories
- one-off compare outputs
- downloaded model files
- copied datasets

If something is generated, default it into `outputs/`.
